All three must hold: the file `/etc/sudoers.d/deployers` exists at mode `440` (check `stat -c '%a'` — sudo ignores anything looser), it passes `visudo -cf /etc/sudoers.d/deployers`, and it contains a `%deployers` rule naming specific command path(s) rather than a bare `ALL`. Command paths must be absolute (start with `/`). If `visudo -cf` reports a parse error, the usual culprits are a missing `=` in the `host=(runas) commands` triple, a lowercase group name mismatch, or a stray line — fix and re-validate. A syntactically valid but blanket `%deployers ALL=(ALL) NOPASSWD: ALL` will be rejected on purpose; narrow it to the command(s) you actually need.
