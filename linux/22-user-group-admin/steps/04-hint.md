The check wants three things at once: `/srv/app` group = `deployers`, mode exactly `2775` (symbolic `drwxrwsr-x` ‚Äî note the `s`, not `x`), and at least one file or subdirectory inside that inherited the `deployers` group. If your mode reads `775` (no `s`), you set `775` instead of `2775` ‚Äî re-run `chmod 2775 /srv/app`. If an existing file inside still shows the wrong group, it was made before you set setgid; delete it and create a fresh one, or `chgrp deployers` it. Setgid only affects files created *after* the bit is set.
