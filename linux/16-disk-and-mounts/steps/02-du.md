# du: what's using the space?

`df` tells you a filesystem is full; `du` (disk usage) tells you *what's*
filling it. The everyday incantation:

```bash
du -sh /data/*
```

- `-s` summarize (one total per argument, not every file)
- `-h` human-readable

To rank subdirectories biggest-first:

```bash
du -sh /data/* | sort -rh
```

`sort -rh` sorts human-readable sizes in reverse. The top line is your
culprit directory. Write its name:

```bash
du -sh /data/* | sort -rh | head -1 | awk '{print $2}' > /tmp/big_dir
cat /tmp/big_dir
```

Real incident pattern: `du -sh /var/* | sort -rh | head` on a full box
almost always points straight at runaway logs or a cache.
