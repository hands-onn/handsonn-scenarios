# df: how full are the filesystems?

`df` (disk free) reports usage per mounted filesystem. `-h` gives
human-readable sizes.

```bash
df -h
```

Columns: Filesystem, Size, Used, Avail, Use%, Mounted on. The one to
watch is **Use%** — a filesystem at 100% causes writes to fail and apps
to crash in confusing ways.

`df -i` shows *inode* usage instead of bytes — you can run out of inodes
(too many tiny files) even with space left, another classic "disk full
but df shows space" gotcha.

When you've looked:

```bash
touch /tmp/df_seen
```
