If `kubectl autoscale ... --cpu-percent=50` was used, the target is already set — just read it back. `autoscaling/v2` objects put it at `.spec.metrics[?(@.resource.name=="cpu")].resource.target.averageUtilization`; `autoscaling/v1` objects put it at `.spec.targetCPUUtilizationPercentage`. Either satisfies this step. `kubectl get hpa cpu-worker -o yaml` shows you which API version your object uses.
