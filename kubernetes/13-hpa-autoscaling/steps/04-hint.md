`kubectl describe hpa cpu-worker` is the fastest read — the `Conditions` and `Events` sections tell you `ScalingActive` true/false and whether it hit `FailedGetResourceMetric`. Then run `kubectl top pods -l app=cpu-worker`; if it errors, metrics-server is absent. The verify only needs `.status.currentReplicas` to be populated (it always is once reconciled) and a non-empty `/tmp/metrics-status` file — write whichever outcome you actually saw.
